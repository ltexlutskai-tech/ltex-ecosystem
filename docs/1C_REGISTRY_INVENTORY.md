> Ціль: відтворити у себе блок «Довідники та регістри» (як у 1С) — зберігати дані в окремих
> регістрах і будувати з них звіти. Цей документ — повний інвентар об'єктів 1С зі статусом
> у нашій системі: ✅ є · 🟡 частково · ❌ нема. Джерела: `docs/1c-export-mobile-full/`
> (мобільний застосунок агента — наш основний скоуп) + `docs/1c-export-2026-06-02/`
> (центральна УТ — звідки беремо фінансові документи й аналітичні регістри).
>
> Зібрано: сесія 5.5 (2026-06-17), 3 паралельні дослідницькі агенти.

## ЧАСТИНА B. Центральна УТ — фінансові документи

> Касові/банківські документи у центральній — «пласкі» (одна сума, без ТЧ товарів) і рухають
> багато бухгалтерських регістрів. Нам цікавий лише борг (`ВзаиморасчетыСКонтрагентами`) +
> каса/банк (`ДвиженияДенежныхСредств`). Решта — податковий облік ФОП, який ми НЕ ведемо.

### B1. Каса (готівка)

| Документ                                  | Статус                                     |
| ----------------------------------------- | ------------------------------------------ |
| Приходний касовий ордер (ПКО)             | ✅ `MgrCashOrder` (income)                 |
| Розходний касовий ордер (РКО)             | ✅ `MgrCashOrder` (expense, `changeForId`) |
| Оплата платіжною карткою                  | 🟡 `amountUahCashless` (без типу «картка») |
| Внутрішнє переміщення наліку (каса→каса)  | ❌                                         |
| Оголошення на внесок готівкою (каса→банк) | ❌                                         |
| Грошовий чек (банк→каса)                  | ❌                                         |

### B2. Банк (безготівка)

| Документ                                            | Статус                                       |
| --------------------------------------------------- | -------------------------------------------- |
| Платіжне доручення вхідне (надходження від клієнта) | 🟡 тоне в `amountUahCashless` без реквізитів |
| Платіжне доручення вихідне (оплата постачальнику)   | ❌                                           |
| Платіжний ордер: надходження/списання за випискою   | ❌                                           |
| Заявка на витрачання коштів / Планове надходження   | ❌ (казначейство — не треба)                 |

### B3. Взаєморозрахунки / борг

| Документ                     | Статус                                                       |
| ---------------------------- | ------------------------------------------------------------ |

| Акт звірки взаєморозрахунків | ❌ (звіт-звірка боргу за період)                             |
| Рахунок на оплату покупцю    | 🟡 друк «Рахунок» з `Order` (нема документа)                 |

## 📊 ПІДСУМОК: головні прогалини для звітів власника

1. **ПродажиСебестоимость** ❌ — без собівартості **неможлива маржа/валовий прибуток** (найцінніший відсутній звіт). Історичної собівартості у базі немає взагалі.
2. **КурсыВалют `_InfoRg4655`** ❌ — дешевий імпорт у `ExchangeRate`, закриває «історичний курс».
3. **Регістри-обороти Продажи / ДенежныеСредства / Залишки** 🟡 — зараз усе on-the-fly агрегацією; для важких звітів за період варто матеріалізувати у власні регістри.
4. **Повернення від покупця** ❌ — єдиний складсько-фінансовий документ, що прямо коригує борг.
5. **Акт звірки боргу** ❌ — простий read-only звіт по `MgrDebtMovement` (дешево, дуже корисно).
6. **Перепаковка** ❌ — специфіка секонд-хенду (розбір мішків), впливає на залишки/собівартість.
7. **Довідник одиниць виміру** ❌, **тайм-трекінг дня агента (СтатусДня)** ❌ — дрібніші.

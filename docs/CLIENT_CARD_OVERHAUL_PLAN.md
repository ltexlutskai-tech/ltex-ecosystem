# План переробки картки клієнта (M3.x)

Зведення 4 досліджень (1С-форма Контрагента + реєстр `РаботаСКлиентом`/авто-записи +
історія продаж/асортимент/презентації/обміни + інвентар нашого коду). Мета — довести
картку клієнта до 1С-паритету + розширення за побажаннями user.

## Класифікація побажань

**A. Локальне — робимо зараз:**

1. Футер: +«Створити реалізацію» (`/manager/sales/new?clientId=`) +«Створити оплату» (`/manager/payments/new?clientId=`).
2. **Телефони** — редагований список (`MgrClientPhone` уже є): додати/видалити, тип месенджера, **справжні лого** (Viber/Telegram/WhatsApp SVG) + кнопки tel/sms/viber/wa.
3. **«Соцмережі та месенджери»** (перейм. з «Соц мережі») — усі інтернет-контакти/посилання клієнта (`MgrClientMessenger`), кожен з іконкою; **дублювати іконками у блок «Контакти»** на шапці картки.
4. **Маршрути** — редаговане призначення (`MgrClientRouteAssignment` ↔ `MgrRoute`): додати/прибрати/порядок. Сам довідник `MgrRoute` наповнюється з 1С (sync — на етапі обмінів).
5. **«Ключові слова»** — нова вкладка/поле: теги клієнта (вручну) → фільтр/пошук у списку клієнтів.
6. **Історія + авто-записи** — `MgrClientTimelineEntry`: відтворити авто-записи 1С по **наших** подіях (замовлення/реалізація/оплата/бронь/нагадування/закриття) з тими ж текст-шаблонами + ручний запис + редагування.
7. **Банк-рахунок → поле в «Дані»** — прибрати окрему вкладку, показати основний рахунок read-only в «Дані» (значення з 1С).

**B. Залежить від 1С-обміну (відкласти дані / можливо stub):**

- **Асортимент** (що купував) = центр. регістр `АсортиментПокупателя` → синк. У нас `MgrClientAssortmentItem` (наповнюється при обміні).
- **Історія продаж** (продані лоти) = центр. `РеализацияТоваровУслуг` (вікно 3 дні). Локально — лише наші `Sale` (часткова).
- **Презентації / Іст. презентацій** = центр. регістр `Презентації` (запит по періоду) + нема модуля презентацій у нас.
- **Маршрути-довідник**, **банк-значення** — sync з центральної.

**C. Greenfield (окремий проєкт):**

- **Об'єднаний чат** (Viber/Telegram/Instagram/WhatsApp/TikTok) — нова фіча, у 1С лише простий Viber-перегляд. Потрібні бот-інтеграції/вебхуки + UI inbox. Проектуємо окремо.

## 1С авто-записи історії (`РаботаСКлиентом`) — каталог (відтворюємо локально по наших подіях)

Реєстр: `Клиент` + `Запись`(текст) + `УИД_Записи` + `АвтоматичнийЗаписБезВзаємодіїЗКлієнтом`.
Кеш на контрагенті: `РаботаСКлиентомПериод/Запись`. Історія **локальна** (не вивантажується).

| Тип                | Тригер                   | Авто?            | Початок тексту                                            |
| ------------------ | ------------------------ | ---------------- | --------------------------------------------------------- |
| Замовлення         | створення замовлення     | ні (дія менедж.) | `Замовлення\n\n<ім'я>\n<обл>\n<місто>\n<тел>…`            |
| Реалізація         | створення реалізації     | ні               | `Реалізація\n\n<Viber-текст>`                             |
| Оплата             | створення оплати         | **так**          | `Оплата\n\n<Viber-текст>`                                 |
| Презентація        | презентація              | **так**          | `<Viber-презентація>`                                     |
| Реал. з МЛ         | реалізація з маршрутного | ні               | `Реалізація\n\n…`                                         |
| Бронь              | встановлення броні       | **так**          | `Встановлення броні\n\n<товар> вага <X> кг період <дата>` |
| Нагадування        | створення нагадування    | **так**/ні       | `Нагадування\n\n<текст>`                                  |
| Закриття замовлень | закриття старих          | **так**          | `Закриття заказів\n\n<дата>\n<позиції>`                   |
| Ручний             | менеджер пише вручну     | ні               | `<текст>`                                                 |

Наші локальні події (замовлення/реалізація/оплата/бронь/нагадування) → пишемо timeline-запис із тим самим форматом. Документні з 1С (презентація, частина продажів) — на етапі обмінів.

Утиліти `phone.ts`/`social-links.ts` (емодзі-іконки) є.

## Відкриті питання (по вкладках) — для user

1. **Ключові слова:** окремі теги на клієнта (вручну) для пошуку/фільтра клієнтів — ТАК? Чи прив'язані до рядків асортименту?
2. **Контакти/телефони/соцмережі:** телефони і «Соцмережі та месенджери» — окремі секції, чи один об'єднаний блок? Як показувати в «Контакти» на шапці?
3. **Історія:** відтворюємо авто-записи по наших подіях (з 1С-форматами) — ТАК?
4. **Історія продаж:** показувати наші локальні Sale зараз, чи лишити stub до 1С?
5. Презентації — лишаємо до 1С (defer)? Банк → основний рахунок у «Дані» (read-only)? Маршрути — редаговане призначення зараз?

## Фазування (чернетка)

- **Фаза 1 (локальне, швидко):** футер +2 кнопки; банк→«Дані»; видалити окрему банк-вкладку.
- **Фаза 2:** телефони (ред.+лого) + «Соцмережі та месенджери» + блок «Контакти» на шапці.
- **Фаза 3:** маршрути (ред. призначення); «Ключові слова» + фільтр у списку клієнтів.
- **Фаза 4:** Історія — авто-записи по наших подіях + редагування запису.
- **Фаза 5 (1С-залежне):** Асортимент/Іст.продаж/Презентації — на етапі обмінів.
- **Окремо:** об'єднаний чат-inbox (greenfield, окремий план).

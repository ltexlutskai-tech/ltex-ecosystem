# Пропозиція: перехід від «вкладок і вікон» (1С-стиль) до одного зручного вікна

**Дата:** 2026-07-14
**Привід:** власнику незручно працювати в менеджерці через вкладки та окремі вікна.
**Суть:** замінити багатовіконну «1С-у-браузері» модель на сучасний односторінковий інтерфейс (як у KeyCRM / Sitniks / Pipedrive), не переписуючи самі екрани.

---

## 1. Що в нас зараз (аналіз коду)

Менеджерка (`apps/store/app/manager/(workstation)/`) побудована як **MDI — багатовіконний інтерфейс, як у 1С**:

- **Оболонка** (`workstation-shell.tsx`): шапка + **рядок вкладок** (`tab-strip.tsx`) + бічне меню + робоча область.
- **Кожна вкладка — окремий `iframe`** (`iframe-host.tsx`): усередині вантажиться справжня сторінка `/manager/...`. Вкладки зберігаються в пам'яті браузера (`localStorage`), не втрачають стан при перемиканні.
- **«Відкріпити в окреме вікно»** (`detachTab`) — відкриває реальне вікно-попап браузера.
- **«Показати поруч»** (`splitId`) — ділить робочу область навпіл.
- Бічне меню — **~30 пунктів** (`sidebar-links.ts`): Замовлення, Реалізація, Оплати, Маршрут, Клієнти, Прайс, Категорії, Нагадування, Потреби, Закриття, Чат, Месенджер, Довідники/регістри, Фінанси, Звіти, Поступлення, адмін-розділи, Кошик, Налаштування…

**Це технічно акуратно зроблено**, але сама модель = 1С: користувач сам відкриває, закриває, шукає серед вкладок і вікон.

## 2. Чому це втомлює (корінь проблеми)

1. **Ви керуєте вікнами, а не роботою.** Накопичується купа вкладок; треба вирішувати що відкрити/закрити, шукати потрібну серед десятка. Це «менеджмент вікон» замість «зробити справу».
2. **`iframe` дає дрібні збої.** Незручності з оновленням сторінки (F5), кнопкою «назад», глибокими посиланнями, повільніший старт, гірша поведінка на телефоні. Частину цього вже латали (7.3), але це боротьба з симптомами моделі.
3. **Навігація «структурна», а не «за задачею».** Спершу обери модуль → потім список → потім документ. Сучасні CRM ведуть від «що мені треба зробити зараз» (нові ліди, прострочене, задачі дня).
4. **Це відтворює саме 1С** — програму, від якої ми свідомо тікаємо. Дублюючи її вигляд, ми успадкували і її втомливість.

## 3. Як зроблено в лідерів (з нашого дослідження)

Жодна сучасна CRM **не використовує вкладки+iframe**. Спільний патерн:

- **Одне вікно:** зафіксована ліва панель розділів + **одна робоча область**. Перемикання розділів — миттєве (без перезавантаження, без iframe).
- **Запис відкривається на місці:** як **бічна панель (slide-over)** праворуч або окрема сторінка — а не нове вікно ОС. Список під нею лишається — «підглянув картку → закрив → працюєш далі».
- **Швидкий пошук `Ctrl+K`** (командна палітра): один вхід з клавіатури — стрибнути до клієнта/замовлення/лота за номером/іменем/**штрихкодом**.
- **«Робочий стіл» = задачі дня**, а не сітка плиток-модулів: нові ліди, прострочені борги/замовлення, нагадування, останні документи.
- **«Нещодавні»** та хлібні крихти для швидкого повернення.
- **Кілька документів поруч?** Це вже вміє браузер: права-кнопка → «відкрити в новій вкладці». Не треба будувати власну систему вкладок і вікон.

## 4. Пропозиція — «Простий режим»: одне вікно замість вкладок

Головна ідея: **прибрати рядок вкладок і iframe**; лишити ліву панель + одну робочу область зі звичайною миттєвою навігацією. Записи відкривати бічною панеллю. Додати `Ctrl+K` і задачний робочий стіл.

### Чому це недорого й безпечно

**Кожна наша вкладка вже є справжньою робочою сторінкою** (`/manager/orders`, `/manager/sales/[id]` тощо) — iframe лише «обгортає» їх. Тобто ми **знімаємо обгортку**, а не переписуємо екрани. Ризик низький, бо самі сторінки не змінюються.

### Що НЕ втрачаємо

- **Кілька документів поруч** — нативні вкладки браузера + бічна панель для «підглянути».
- **Швидке перемикання** — `Ctrl+K` + «Нещодавні» замість пошуку серед вкладок.
- **Звичну логіку блоків** — ті самі розділи в лівій панелі, ті самі списки й картки.

### Етапи (низький ризик, можна відкотити будь-коли)

**Етап 1 — новий каркас + перемикач режиму.**

- Новий shell без iframe/вкладок: ліва панель + контент, звичайна клієнтська навігація Next.js.
- Перемикач у налаштуваннях **«Класичний (вкладки) / Простий (одне вікно)»** — щоб спробувати без ризику й порівняти. За замовчуванням поки класичний.
- _Технічно:_ рендерити ті самі маршрути напряму (прибрати `IframeHost`/`TabStrip` за прапорцем), звичайні `<Link>` замість `openTab`.

**Етап 2 — зробити роботу приємною.**

- **Бічна панель (slide-over)** для карток: клік по рядку списку → картка виїжджає праворуч, список видно.
- **`Ctrl+K`** — швидкий пошук/стрибок за номером, іменем, штрихкодом.
- **Задачний «Робочий стіл»** — нові ліди, прострочене, нагадування, останні документи + «Нещодавні».
- Впорядкувати ліву панель (30 пунктів → згруповані розділи, що згортаються).

**Етап 3 — зробити «Простий» типовим.**

- Після того як режим сподобається — увімкнути його за замовчуванням; класичний лишити опційним для тих, хто звик, або прибрати.

## 5. Рекомендація

Почати з **Етапу 1 + бічної панелі та `Ctrl+K` з Етапу 2** — це дає одразу відчутне полегшення («одне вікно, нічого не треба закривати»), а перемикач робить перехід безпечним. Це доповнює блок «UI-освіження» з `docs/CRM_IMPROVEMENT_ROADMAP.md` і не конфліктує з рештою плану.

**Візуальний макет нового вигляду** — див. артефакт цієї сесії (клікабельний прототип: ліва панель + список + бічна картка + `Ctrl+K`).

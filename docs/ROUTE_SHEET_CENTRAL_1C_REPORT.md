## 0. Суть одним абзацом

**Маршрутний лист = документ рейсу.** Він веде весь цикл в одному місці: які **замовлення** їдуть у рейс → який **товар потрібен** під ці замовлення → що **фактично завантажили** в машину (мішок за мішком) → які **продажі** й **оплати** зробили в клієнтів → **закриття** рейсу зі списанням товару зі складу. Це не просто список замовлень — це водночас **складський документ завантаження** і **касово-збутовий документ рейсу**.

## 2. Головний екран (те, що на ваших скрінах)

Форма розбита на три робочі зони:

1. **Ліворуч — «Заказы»** (перемикач **Дерево / Список**). Дерево дворівневе:
   батько = замовлення (клієнт), діти = товари цього замовлення. Колонки:
   `Місто · Контрагент · Артикул · Номенклатура · Вес · Единица · Заказано · Факт · Загружено · Осталось · Ціна · Сума`.
   Кнопки: **Подбор · Заповнити · Проверить остатки · Обновить количества · По заказу**.

2. **Праворуч зверху — «Загрузка машины»** — таблиця фактично завантажених мішків
   (N, Заказ, Артикул, Номенклатура, Характеристика, Штрихкод, Кількість, Ціна…).
   Кнопки: додати/редагувати/видалити рядок, ↑↓, сортування, **сканер ШК**, **Заповнити**.

3. **Праворуч знизу — «Продажи»** + кнопка **«Перепровести всі реалізації»**.

Унизу — лічильник: **«Заказов: 20; заказано: 85; загружено: 21»**.

## 3. Колонки дерева — точні формули

| Колонка         | Звідки береться                                                                                                  |
| --------------- | ---------------------------------------------------------------------------------------------------------------- |
| **Заказано**    | `ТоварыЗаказов.Количество` — планова кількість із замовлення клієнта                                             |
| **Факт**        | `ТоварыЗаказов.КоличествоФакт` — уточнена фактична потреба                                                       |
| **Загружено**   | `ТоварыЗаказов.КоличествоЗагружено` — **сума завантаженого** з таблиці ЗагрузкаМашины по цьому замовленню+товару |
| **Осталось**    | **має ДВА значення** залежно від натиснутої кнопки (див. нижче)                                                  |
| **Ціна / Сума** | з `ТоварыЗаказов`                                                                                                |

**«Осталось» — важливий нюанс (це і збивало нас раніше):**

- Після кнопки **«Обновить количества»**: `Осталось = Заказано − Загружено` (скільки ще довантажити).
- Після кнопки **«Проверить остатки»**: `Осталось = ВільнийЗалишокСкладу − Загружено`, де вільний залишок = **`ТоварыНаСкладах − ТоварыВРезервеНаСкладах`** з урахуванням броні мішків.

Саме тому на скрінах видно «Заказано 1, Загружено 1, **Осталось 4**» — це показ **вільного залишку складу** (ще 4 вільні мішки цього товару лежать), а не «замовлено−завантажено».

**Підсвітка рядків** (те, що бачить склад): зелене — завантажено повністю; жовте — частково (прогрес); червоне — товару немає на вільному залишку.

## 4. Кнопки і що кожна робить — це і є ПРОЦЕС

### Панель «Заказы (дерево)»

- **Подбор** — відкриває вибір замовлень (можна кілька). Для кожного: додає рядок у `Заказы` + **переносить усі товари замовлення в `ТоварыЗаказов`** (потреба рейсу) + одразу «Проверить остатки» + перебудова дерева. **Це головний спосіб набити рейс замовленнями та потребою.**
- **Заповнити товар** — перезаповнює товари для **поточного** замовлення дерева з його замовлення-джерела.
- **Заповнити всі товари** — те саме для **всіх** замовлень рейсу.
- **Обновить количества** — перераховує Загружено/Осталось із таблиці ЗагрузкаМашины (Осталось = Заказано − Загружено).
- **Проверить остатки** — запитує **вільний залишок складу** (ТоварыНаСкладах − резерви + бронь мішків) і кладе в «Осталось» + «не хватает».
- **По заказу / Отбор загрузки** — фільтр рядків завантаження по обраному замовленню.

### Панель «Загрузка машины»

- **Поиск по штрихкоду (скан)** — головний спосіб завантаження: сканує ШК мішка → знаходить товар+мішок → додає рядок у `ЗагрузкаМашины` і прив'язує до замовлення.
- **Заповнити** — очищає завантаження й генерує **по одному рядку на кожну одиницю** потреби (Кількість=1, з ШК), пропускаючи заброньовані.
- **Загрузка в заказ** — перемикач: коли ввімкнено, наступний сканований товар йде в **поточне** замовлення дерева; коли вимкнено — система сама шукає перше недовантажене замовлення під цей товар.
- **Загрузка / возврат** — перемикач режиму «повернення» (мінусовий рядок замість завантаження).
- **Изменить состояние возврата** / **Удалить строку** / **Печать ШК**.
- ⚠️ **Ручне «додати рядок» у Загрузку заборонено** — рядки з'являються лише через скан або кнопку «Заповнити» (щоб кожен мішок був реальним, з ШК і вагою).

### Прив'язка «завантажено → замовлення»

За таким пріоритетом: **1)** за точним **Штрихкодом**; **2)** за **Номенклатура + Характеристика(мішок)** до першого недовантаженого замовлення; **3)** у поточне замовлення дерева (режим «Загрузка в заказ»). Серія/лот у прив'язці не бере участі — ключ це **товар + мішок(характеристика) + штрихкод**.

## 5. Повний процес «від замовлень до завантаженої машини»

1. **Логіст** створює маршрутний лист, тисне **«Подбор»**, обирає замовлення клієнтів у рейс → їх товари падають у **ТоварыЗаказов** (потреба). Статус **Складається**.
2. **«Проверить остатки»** — дерево показує по кожному товару: Заказано / вільний залишок складу / бронь / чого не хватає.
3. **Завантаження складом**: комірник **сканує ШК мішків** (або тисне «Заповнити»). Кожен мішок → рядок **ЗагрузкаМашины** (вага, ціна, штрихкод), прив'язаний до замовлення. Заброньований чужим агентом мішок сканувати не можна.
4. **«Обновить количества»** — Загружено/Осталось у дереві оновлюються; підсвітка показує готовність.
5. Коли все набрано — статус **Відправлений**: документ проводиться, товар **списується зі складу** (`ТоварыНаСкладах` розхід + зняття резерву) і стає **«у дорозі»** (`ТоварыВДороге` прихід).
6. **У клієнта**: агент по замовленню створює **Реалізацію** → продані мішки списуються із ЗагрузкаМашины і фіксуються в **Продажи**; далі **Оплата** (касовий ордер).
7. Статус **Завершений**: товар «доїхав» (`ТоварыВДороге` розхід датою приїзду), рейс закривається й блокується від змін; замовлення позначаються неактуальними.

## 6. Залишок і бронь (складська логіка)

- **Вільний залишок** = `ТоварыНаСкладах.Остаток − ТоварыВРезервеНаСкладах.Остаток` (регістр резервів під замовлення).
- **Бронь мішка** — на рівні характеристики (мішка): реквізити `Бронь` (агент), `Контрагент`, `ПериодБрони`. Заброньований мішок не вільний і не сканується — крім агента зі списку `ТорговыеАгенты` документа.
- «Не хватает» = потреба (Заказано/Факт − Загружено) мінус вільний залишок складу.

> У нашій системі аналог: `Lot` зі `status='free'` + денормалізована бронь (`reservedByUserId/reservedUntil`). Реєстру резервів `ТоварыВРезервеНаСкладах` у нас окремо немає — «вільний лот» уже виключає зайняті.

## 7. Статуси і проведення (що змінюється на складі)

| Статус           | Що відбувається з рухами                                                                                                                                                                                                       |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Складається**  | рухів немає (тільки планування/завантаження в документі)                                                                                                                                                                       |
| **Відправлений** | розхід `ТоварыНаСкладах` + `ТоварыВРезервеНаСкладах` + `ТовариНаСкладахУВазі` (у вазі) + `ПартииТоваров`; прихід `ТоварыВДороге`; рядки `Возврат` — навпаки прихід на склад; витрати → розхід каси + `ДвиженияДенежныхСредств` |
| **Завершений**   | `ТоварыВДороге` розхід (датою приїзду) — товар доїхав; рейс закривається                                                                                                                                                       |

Регістри-реєстратори: `ТоварыНаСкладах, ТоварыВРезервеНаСкладах, ПартииТоваровНаСкладах, ТоварыОрганизаций, ТоварыВДороге, ТовариНаСкладахУВазі, Возвраты, ДенежныеСредства, ДвиженияДенежныхСредств`.

## 8. Десктоп (центральна) vs мобільний агент

- **Той самий документ** існує в обох конфігураціях.
- **Центральна (десктоп)** — форма з ваших скрінів: повний набір (Подбор, Проверить остатки, Обновить количества, редагована Загрузка, скан). Тут логіст планує рейс, а склад завантажує.
- **Мобільний агент** — у полі: сканує ШК на завантаженні, у клієнта створює реалізації/оплати. Дані синхронізуються назад у центральну (`ОбменАРМ`).

## 9. Чому наша поточна «Загрузка» не дозволяє наповнити маршрутник — і що треба

**Що ми зробили зараз (недостатньо):** тільки поле скану одного ШК + «сліпе» авто-заповнення + read-only дошка. Це вузький шматок мобільного сценарію, без десктопного інструментарію складу.

**Чого бракує проти центральної бази:**

1. **Редагована таблиця «Загрузка машины»** — можливість додавати/міняти рядки завантаження (мішок, кількість, вага, ціна), а не лише сканувати по одному.
2. **«Подбор»** — набір рейсу замовленнями + перенесення їх товарів у потребу (у нас це «Додати замовлення» + «Заповнити товари», але без єдиного зручного підбору й перерахунку).
3. **Дві окремі колонки залишку** в дереві: **вільний залишок складу** та **бронь**, плюс «не хватает».
4. **Кнопки «Проверить остатки» / «Обновить количества»** — щоб склад бачив живу картину (замовлено / завантажено / вільно на складі / не вистачає).
5. **Режими «Загрузка в заказ» і «Возврат»** — прив'язка сканованого до конкретного замовлення та завантаження-повернення.
6. **Прив'язка завантаженого до замовлення за пріоритетом ШК → товар+мішок → поточне замовлення** (у нас спрощено — тільки за товаром).

**Рекомендована модель для нашої системи:**

- Екран завантаження = **робоча таблиця «Загрузка машини»** (додавання/редагування/видалення рядка, скан ШК, режими «в заказ»/«возврат») **поруч** із **деревом замовлень** (Замовлено / Завантажено / Вільний залишок / Бронь / Не хватає, з підсвіткою).
- Кнопки: **Підбір замовлень**, **Заповнити потребу**, **Перевірити залишки**, **Оновити кількості**.
- Прив'язка мішка до замовлення за штрихкодом → товар+лот → поточне/перше недовантажене замовлення.
- Все на наших даних (`RouteSheetItem` = ТоварыЗаказов, `RouteSheetLoading` = ЗагрузкаМашины, лоти = мішки/характеристики). Без 1С.

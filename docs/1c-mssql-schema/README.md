# Дамп схеми 1С MSSQL — reference для сесії 5.2 (імпорт історії)

Цю папку наповнює `scripts/dump-1c-mssql-schema.ps1` (запускати на Windows-сервері,
де крутиться 1С).

## Файли

- `tables.tsv` — усі таблиці бази `ltex` з кількістю рядків і розміром (МБ).
- `columns.tsv` — повний список колонок (`INFORMATION_SCHEMA.COLUMNS`).
- `indexes.tsv` — індекси з переліком колонок (для розуміння PK/FK).
- `prefix_summary.tsv` — зведення таблиць за типом метаданих 1С
  (`_Reference*` = довідники, `_Document*` = документи, тощо).
- `config_tables.tsv` — список системних таблиць `Config/Params/IBVersion`
  які зберігають декодер імен метаданих 1С.

## Як декодувати імена

1С автоматично перейменовує таблиці у `_Reference77`, `_Document108`, тощо.
Реальні назви (наприклад «Контрагенти», «Замовлення покупця») зберігаються:

1. У бінарному blob-і таблиці `Config` / `_Config` — _не читається_ напряму
   через SQL (запакований у власний LZF-формат 1С).
2. У **експорті конфігурації** який вже лежить у `docs/1c-export-2026-06-02/`
   (зроблений 2026-06-02 з 1С Конфігуратора). Там `Catalogs/Контрагенти.xml`,
   `Documents/ЗаказПокупателя.xml` і т.д. — стандартні ASCII-назви.

Стратегія мапування для сесії 5.2:
**не лізти у Config blob**, замість того співставити структуру колонок
`_Reference77` зі структурою у `docs/1c-export-2026-06-02/Catalogs/<X>.xml`
(збігаються UUID-и реквізитів + кількість колонок + типи).

## Запуск

```powershell
cd E:\ltex-ecosystem
.\scripts\dump-1c-mssql-schema.ps1
git add docs/1c-mssql-schema/
git commit -m "docs: дамп схеми 1С MSSQL для сесії 5.2"
git push origin main
```

Скрипт читає пароль з `apps/store/.env::LEGACY_1C_DB_URL` —
у git нічого секретного не потрапляє.

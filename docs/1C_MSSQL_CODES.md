# Декодовані коди MSSQL 1С (офлайн) — довідка для звірки

Декодовано **офлайн** з дампу схеми `docs/1c-mssql-schema/` (без живої БД):

1. `<метадані-назва>.xml` (у `docs/1c-export-2026-06-02/` або `docs/1c-export-mobile-full/`)
   → витягнути `uuid="…"` об'єкта.
2. `dbnames.txt` → UUID → фізична таблиця (`_AccumRgNNNN` / `_DocumentNNN` /
   `_ReferenceNNN` / `_InfoRgNNNN`).
3. `columns.tsv` → перелік колонок таблиці. Зіставити порядок оголошення
   вимірів/ресурсів/реквізитів з XML; кожен простий реквізит = 1 `_FldNNNN`,
   кожен поліморфний = трійка `_TYPE/_RTRef/_RRRef`. Типи (`numeric`/`binary`/
   `datetime`/`nvarchar`) звірені з декларацією XML → дають однозначну відповідність.
4. `indexes.tsv` — підтверджує виміри (вони індексовані `ByDimsNNN`).

**Confidence:** HIGH = точний позиційний+типовий збіг; усе нижче — HIGH, якщо не
зазначено інакше. Жодного запиту до живої БД не виконано — лише редагування коду.

> ⚠️ Entity лишаються ПОЗА `DEFAULT_ORDER`. User звіряє `--dry-run` / `--confirm-prod`
> перед записом. Де офлайн-дамп неоднозначний — у коді лишено `// TODO(LOW)`.

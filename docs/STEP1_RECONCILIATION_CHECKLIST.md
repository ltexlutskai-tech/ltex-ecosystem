# Крок 1 — Чек-лист звірки перенесених даних (наша система ↔ 1С)

> Мета: переконатися, що цифри в новій системі збігаються з 1С, перш ніж офіційно
> відключати 1С. Проходимо по пунктах; для кожного — стан + що робити.
> **Порада:** оберіть ОДИН фіксований період для звірки (напр. конкретний закритий
> місяць — травень 2026, або весь 2025 рік) і скрізь порівнюйте той самий період.

Колонка «Стан» — заповнює власник: ✅ збігається / ⚠️ розбіжність (написати яку) / ❔ не зрозуміло.

## A. Швидка перевірка «чи взагалі залилось» (5 хв, без 1С)

Відкрити в менеджерці → **Довідники та регістри** і переконатися, що переглядачі НЕ порожні:

| #   | Переглядач регістру | Очікування                    | Стан |
| --- | ------------------- | ----------------------------- | ---- |
| A1  | Продажі             | є рядки (товар/клієнт/сума)   |      |
| A2  | Рух коштів (ДДС)    | є рядки (рахунок/стаття/сума) |      |
| A3  | Залишки товарів     | є рядки (склад/товар/кг)      |      |
| A4  | Залишки замовлень   | є рядки                       |      |
| A5  | Курси валют         | є ряд курсів по датах         |      |
| A6  | Борг (рухи)         | є рядки (вже звіряли раніше)  |      |

→ Якщо якийсь ПОРОЖНІЙ — той регістр не залився (скажи мені, перевіримо імпорт).

## B. Звірка цифр із 1С (головне) — по одному періоду

### B1. Борг клієнтів (найважливіше)

- **У нас:** Звіти → Прострочені борги (загальна сума) + картка 2-3 конкретних клієнтів (поле «Борг»).
- **У 1С:** «Ведомость взаиморасчётов з контрагентами» / борг по тих самих клієнтах на ту саму дату.
- **Звіряємо:** (а) борг 2-3 іменних клієнтів; (б) загальна сума дебіторки.
- **Стан:**

### B2. Продажі — виручка і кг за період

- **У нас:** Звіти → Підсумок продажів (за обраний місяць): загальна сума + загальна вага (кг) + топ-агент.
- **У 1С:** звіт «Продажи» (Обороти) за той самий місяць.
- **Звіряємо:** виручку за місяць; вагу (кг) за місяць; суму по 1-2 торгових агентах.
- **Стан:**

### B3. Маржа / валовий прибуток

- **У нас:** Звіти → Маржа (за період): Виручка − Собівартість = Валовий прибуток.
- **У 1С:** «Валовая прибыль» за той самий період.
- **Звіряємо:** валовий прибуток за місяць; маржа по 1-2 топ-товарах.
- **Стан:**

### B4. Рух коштів (каса/банк, ДДС)

- **У нас:** Звіти → Рух коштів (за період): прихід / розхід / сальдо по статтях.
- **У 1С:** «Движения денежных средств» за той самий період.
- **Звіряємо:** загальний прихід і розхід за місяць; 1-2 найбільші статті.
- **Стан:**

### B5. Залишки складу

- **У нас:** Звіти → Залишки складу (на конкретну дату): загальні штуки + кг.
- **У 1С:** «Остатки товаров» / «Товары на складах» на ту саму дату.
- **Звіряємо:** загальну вагу (кг) і штуки на дату.
- **⚠️ Відомий нюанс:** історія складських документів (повернення/перепаковки/списання)
  ще НЕ перенесена — тож залишки за минулі періоди можуть трохи розходитись. Якщо розбіжність
  невелика й саме «на складських рухах» — це очікувано, доробимо переносом цих документів.
- **Стан:**

### B6. Курс валют на дату

- **У нас:** Довідники та регістри → Курси валют (знайти курс EUR на конкретну дату).
- **У 1С:** курс EUR на ту саму дату.
- **Звіряємо:** збіг курсу (щоб грн-суми рахувались правильно).
- **Стан:**

### B7. Кількості документів/записів за період

- **У нас:** списки Замовлення / Реалізація / Оплати (фільтр за період) — кількість.
- **У 1С:** кількість тих самих документів за період.
- **Звіряємо:** к-сть замовлень; к-сть реалізацій; к-сть касових ордерів; к-сть клієнтів (~5000).

## C. Що робимо за результатами

- **✅ збігається** → пункт закритий.
- **⚠️ розбіжність** → опиши: який звіт, який період, наша цифра vs 1С. Я знайду причину
  (найімовірніше — код колонки в імпорті, як було з cashflow/orders/bankdocs) і виправлю.
- **Систематична розбіжність на складських рухах** → очікувано (не перенесено повернення/
  перепаковки); план — дописати їх перенос (Крок 1, частина 2).

Коли більшість пунктів ✅ — 1С можна офіційно перевести в режим «тільки довідка».

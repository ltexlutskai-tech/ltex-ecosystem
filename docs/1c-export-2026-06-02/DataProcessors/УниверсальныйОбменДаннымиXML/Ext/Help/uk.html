Обробка "Універсальний обмін даними у форматі XML" призначена для завантаження й вивантаження даних у файл із будь-якої конфігурації, реалізованої на платформі 1С:Підприємство 8

Обробка має чотири закладки

Вивантаження даних

Для здійснення вивантаження даних необхідно вказати ім'я файлу, у який буде здійснюватися вивантаження даних і вибрати файл правил обміну. Правила обміну для будь-яких конфігурацій можуть бути настроєні в спеціалізованій конфігурації "Конвертація даних, редакція 2".

Для вивантаження документів і записів незалежних періодичних регістрів відомостей необхідно вказати період - "Дату початку" і "Дату закінчення". Результуючий файл із вивантаженими даними може бути стиснутий.

На закладці "Правила вивантаження даних" можна вибрати ті типи об'єктів, які повинні вивантажуватися, настроїти відбори для вибірки об'єктів, або вказати вузол обміну даними, для якого потрібно вивантажувати дані.

На закладці "Параметри вивантаження" можна вказати додаткові параметри вивантаження даних.

На закладці "Коментар" можна написати довільний текст- коментар, що включається у файл обміну.

Завантаження даних

Для здійснення завантаження даних необхідно вказати ім'я файлу, з якого буде здійснюватися завантаження даних.

Є можливість настроїти завантаження даних у транзакції. Для цього необхідно встановити прапорець "Використовувати транзакції" і вказати кількість елементів в одній транзакції при завантаженні.

"Завантажувати дані в режимі обміну (ОбменДанными.Загрузка = Истина)" - якщо прапорець встановлений, то завантаження об'єктів буде виконуватися із установленою ознакою завантаження. Це означає, що при записі об'єктів у базу даних будуть відключені всі платформні й прикладні перевірки. Виключення становлять документи, які записуються в режимі проведення або скасування проведення. Проведення й скасування проведення документа виконується завжди без установки режиму завантаження, тобто перевірки будуть виконуватися.

Додаткові настройки

Закладка служить для детальної настройки вивантаження й завантаження даних.

"Режим налагодження" - прапорець для завдання режиму налагодження обміну. Якщо цей прапорець встановлений, то процес обміну даними не буде зупинений при виникненні якої-небудь помилки. Обмін завершиться до кінця з виведенням повідомлень налагодження у файл протоколу обміну. Цей режим рекомендується використовувати при налагодженні правил обміну.

"Виведення інформаційних повідомлень у вікно повідомлень" - якщо прапорець установлений, то у вікно повідомлень буде виводитися протокол процесу обміну даними.

"Кількість оброблених об'єктів для відновлення статусу" - параметр служить для визначення кількості оброблених елементів перед зміною рядка стан завантаження/вивантаження

"Настройки вивантаження даних" - дозволяють визначити кількість елементів, що оброблюються в одній транзакції при вивантаженні даних, вивантажувати й обробляти тільки ті об'єкти, на які є права доступу, настроїти тип зміни реєстрації для вивантажених об'єктів  через плани обміну.

"Використовувати оптимізований формат для обміну даними (V8 - V8, версія обробки не нижче 2.0.18)" - оптимізований формат повідомлення обміну вимагає наявність вузла "ИнформацияОТипахДанных" у заголовку повідомлення, у який вивантажується інформація про типи даних. Це дозволяє прискорити процес завантаження даних.

"Використовувати транзакції при вивантаженні для планів обміну" - прапорець визначає режим використання транзакцій при вивантаженні даних при вибірці змін на вузлах планів обміну. Якщо прапорець установлений, то вивантаження даних буде виконуватися в транзакції.

"Кількість елементів у транзакції" - визначає максимальне число елементів даних, які містимуться в повідомленні в рамках однієї транзакції бази даних. Якщо значення параметра дорівнює 0 (значення за замовчуванням), то всі дані містимуться в рамках однієї транзакції. Такий режим є рекомендованим, тому що гарантує актуальність даних, що містяться в повідомленні. Але при створенні повідомлення в багатокористувацькому режимі можуть бути конфлікти блокувань між транзакцією, у якій дані містяться в повідомленні, і транзакціями, виконуваними іншими користувачами. Для зниження ймовірності виникнення таких конфліктів можна задати значення цього параметра, відмінне від значення за замовчуванням. Чим менше значення параметра, тем менше ймовірність конфлікту блокувань, але вище ймовірність приміщення в повідомлення неузгоджених даних.

"Вивантажувати об'єкти на які є права доступу" - якщо прапор установлений, то вибірка об'єктів інформаційної бази буде виконуватися з урахуванням прав доступу поточного користувача програми. Це припускає використання літерала "РАЗРЕШЕННЫЕ" у тексті запиту для вибірки даних.

"Автоматично видаляти неприпустимі символи з рядків для запису в XML" - якщо прапорець установлений, то при записі даних у повідомлення обміну неприпустимі символи будуть вилучені. Символи перевіряються на відповідність рекомендації XML 1.0.

"Зміни реєстрації для вузлів обміну після вивантаження" - поле визначає режим роботи з реєстрацією змін даних після завершення вивантаження даних. Можливі значення:

  • Не видаляти реєстрацію - після вивантаження даних реєстрація змін на вузлі вилучена не буде.
  • Повністю вилучити реєстрацію для вузла обміну - після вивантаження даних реєстрація змін на вузлі буде повністю вилучена.
  • Вилучити реєстрацію тільки для вивантажених метаданих - після вивантаження даних реєстрація змін на вузлі буде вилучена тільки для об'єктів метаданих, які були зазначені до вивантаження.


"Протокол обміну" - дозволяє настроїти виведення інформаційних повідомлень у вікно повідомлень, ведення й запис в окремий файл протоколу обміну.

"Ім'я файлу, протоколу обміну" - ім'я файлу для виведення протоколу процесу обміну даними.

"Протокол завантаження (для COM - з'єднання)" - ім'я файлу для висновку протоколу процесу обміну даними в базі- приймачі при обміні через Com- З'єднання. Важливо: шлях до файлу повинен бути доступний з комп'ютера, на якому встановлена база- приймач.

"Дописувати дані до протоколу обміну"  - якщо прапорець установлений, то вміст файлу протоколу обміну зберігається, якщо файл протоколу вже існує.

"Виведення до протоколу інформаційних повідомлень" - якщо прапорець установлений, то до протоколу обміну будуть виводитися повідомлення інформативного характеру, крім повідомлень про помилки обміну.

"Відкривати файли протоколів обміну після виконання операцій" - якщо прапорець установлений, то після виконання обміну даними файли протоколів обміну будуть автоматично відкриті для перегляду.

Видалення даних

Закладка потрібна тільки для розроблювачів правил обміну. Дозволяє видаляти з інформаційної бази довільні об'єкти.

Налагодження вивантаження й завантаження даних 

Обробка дозволяє робити налагодження оброблювачів подій і генерувати модуль налагодження з файлу- правил або файлу- даних.

Включення режиму налагодження оброблювачів вивантаження проводиться на закладці "Вивантаження даних" установкою прапорця "Режим налагодження оброблювачів вивантаження". Відповідно, на закладці "Завантаження даних" включення режиму налагодження завантаження проводиться установкою прапорця "Режим налагодження оброблювачів завантаження". 

Після установки режиму налагодження оброблювачів стане доступним кнопка настройки налагодження. По натисканню на цю кнопку відкриється вікно настройки.

Настройка налагодження оброблювачів виконується в чотири кроки:

Крок 1: Вибір режиму налагодження алгоритмів


На першому кроці необхідно визначитися з режимом налагодження алгоритмів:

  • Без налагодження алгоритмів
  • Викликати алгоритми як процедури
  • Підставляти код алгоритмів на місці виклику

Перший режим зручно використовувати, коли ми точно знаємо, що помилка в оброблювачі не пов'язана з кодом якого-небудь алгоритму. У цьому режимі код алгоритмів не вивантажується в модуль налагодження. Алгоритми виконуються в контексті оператора "Виконати()" і їх код недоступний для налагодження.

Другий режим необхідно використовувати в тих випадках, коли помилка знаходиться в коді алгоритму. При установці цього режиму алгоритми будуть вивантажені як окремі процедури. У момент виклику алгоритму з якого-небудь оброблювача відбувається звертання до відповідної до процедури обробки. Цей режим зручно використовувати, коли для передачі параметрів в алгоритми використовується глобальна змінна "Параметри". Обмеження використання цього режиму в тому, що при налагодженні в алгоритмі недоступні локальні змінні оброблювача, з якого він викликається.

Третій режим налагодження використовується, як і в другому випадку, при налагодженні коду алгоритмів і в тих випадках, при яких другий режим налагодження не підходить. При установці цього режиму алгоритми будуть вивантажені як інтегрований код в оброблювачах. Тобто замість оператора виклику алгоритму вставляється повний код алгоритму з урахуванням вкладених алгоритмів. У цьому режимі немає обмежень на використання локальних змінних оброблювача, однак є обмеження при налагодженні алгоритмів з рекурсивним викликом.

Крок 2: Формування модуля налагодження


На другому кроці необхідно зробити вивантаження оброблювачів натисканням на кнопку "Сформувати модуль налагодження вивантаження (завантаження)". Сформовані оброблювачі й алгоритми будуть виведені в окреме вікно для перегляду. Уміст модуля налагодження необхідно скопіювати в буфер обміну натисканням на кнопку "Копіювати в буфер обміну".

Крок 3: Створення зовнішньої обробки


На цьому кроці необхідно запустити конфигуратор і створити нову зовнішню обробку. У модуль обробки необхідно вставити вміст буфера обміну (модуль налагодження) і зберегти обробку під будь-яким іменем. 

Крок 4: Підключення зовнішньої обробки


На четвертому, завершальному, кроці, треба вказати ім'я файлу зовнішньої обробки в поле введення. При цьому програма виконує перевірку часу створення (оновлення) файлу обробки. Якщо обробка має більш ранню версію, ніж версія файлу модуля налагодження, то буде виведене попередження й форма настройки закрита не буде.

Примітка: Можливість налагодження глобального оброблювача конвертації "Після завантаження правил обміну" не підтримується.
